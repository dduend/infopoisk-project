## поиск по сюжетам фильмов

поисковая система по сюжетам фильмов с использованием трех типов индексов:
- **BM-25** - классический поиск по точному совпадению слов
- **Word2Vec** - поиск по смыслу (обучение на корпусе)
- **FastText** - семантический поиск с учетом подстрок (обучение на корпусе)

проект реализует CLI-интерфейс для поиска по корпусу из 2000 сюжетов фильмов

автор: Далия Гимадиева

### корпус данных

источник: [Movie Plots from Wikipedia in Russian](https://www.kaggle.com/datasets/maksimpotorochin/movie-plots-from-wikipedia-in-russian). корпус объемом 2000 документов (первые 2000 записей из датасета), содержит сюжеты фильмов из русской википедии

### используемые модели

1. **BM-25**
   - библиотека: `bm25s` (https://github.com/xhluca/bm25s)
   - параметры: k1=1.5, b=0.75, method='lucene'
   - метод `lucene` - реализация BM-25, которая отличается от классической формулы Робертсона-Спарка-Джонса тем, что вводит дополнительную нормализацию частоты термина
   - *p.s. были проблемы с совместимостью библиотек, питона, C+ окружения... поэтому работала с тем, что удалось установить внутри пайчарма*

2. **Word2Vec**
   - библиотека: `gensim`
   - обучение: на корпусе сюжетов фильмов
   - параметры: векторная размерность 100, окно 5, min_count=1
   - модель `skip-gram` - предсказывает контекстные слова по целевому слову, лучше работает с редкими словами, чем альтернативный метод cbow

3. **FastText**
   - библиотека: `fasttext-wheel` (https://fasttext.cc/)
   - обучение: на корпусе сюжетов фильмов
   - параметры: векторная размерность 100, окно 5, эпох 10
   - модель `skipgram` - так же, как и в word2vec, но с ключевым отличием: fasttext учитывает внутреннюю структуру слов, разбивая их на n-граммы, что позволяет получать векторы даже для слов, не встречавшихся в обучении, и лучше работать с морфологически богатыми языками (как русский)

### предобработка текста

все тексты проходят одинаковую предобработку:

1. **очистка:** удаление пунктуации, цифр, специальных символов
2. **приведение к нижнему регистру**
3. **лемматизация:** приведение слов к начальной форме (библиотека `pymorphy3`)
4. **удаление стоп-слов:** стандартные русские стоп-слова + кино-специфичные (фильм, кино, режиссер, актер и тп)
5. **удаление коротких слов:** слова длиной <= 2 символов

### структура проекта
- `main.py` - CLI точка входа
- `examples.py` - демонстрационные примеры: полная (8 примеров) и быстрая демонстрация (3 индекса, 1 запрос)
- `indices.py` - реализации индексов (BM25SIndex, Word2VecIndex, FastTextIndex)
- `preprocessing.py` - предобработка текста
- `requirements.txt` - зависимости
- `README.md` - документация
- `data/films_data.csv.zip` - корпус данных

### запуск

1. **базовый поиск (BM-25, 5 результатов)**
   - `python main.py -q "история любви"`
3. **поиск с указанием индекса и количества результатов**
   - `python main.py -q "детектив убийство" -i bm25 -k 10`
   - `python main.py -q "смешная комедия" -i word2vec -k 5`
   - `python main.py -q "приключения в космосе" -i fasttext -k 3`
3. **демонстрация примеров**
   - `python examples.py` с 3 режимами демонстрации
  
### CLI параметры
- `--query` - сокращение `-q`, реализует поисковый запрос, обязательный параметр
-  `--index` - сокращение `-i`, отвечает за тип индекса: `bm25`, `word2vec`, `fasttext` (по умолчанию bm25), необязательный параметр
-  `--top-k` - сокращение `-k`, отвечает за количество результатов (по умолчанию 5), необязательный параметр

при выполнении поиска выводится время, затраченное на обработку запроса (fasttext обучается один раз при первом запуске, последующие запуски используют уже построенную модель)
